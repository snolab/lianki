## Bugs Found

### BUG-1 (Critical): `/next` redirects to non-existent `/repeat` route

**URL:** `/en/next`
**Symptom:** Navigating to `/en/next` causes a redirect chain that ends at `/ja/repeat?id=…` → 404
**Root Cause:**
- `app/[locale]/next/page.tsx` does `redirect("/api/fsrs/next")`
- `/api/fsrs/next` handler in `app/fsrs.ts:419-437` returns HTML with JS: `location.href='/repeat/?${q}'`
- `/repeat/` is not a Next.js route; the intlayer middleware intercepts it and redirects to `/ja/repeat?id=…`
- `/[locale]/repeat` page does not exist → 404
**Fix:** Change `location.href='/repeat/?${q}'` → `location.href='/api/fsrs/repeat/?${q}'` in `app/fsrs.ts`

---

### BUG-2 (Critical): `/polyglot` crashes with 500 server error

**URL:** `/en/polyglot`
**Symptom:** `Application error: a server-side exception has occurred` (Digest: 4216112987)
**Root Cause:**
- `app/[locale]/polyglot/layout.tsx:21` calls `getUserMembership(user.id)` without a try/catch
- `getUserMembership` throws `Error("User not found")` when the user document is not found in the MongoDB `user` collection via `{ id: userId }` query
- This unhandled error propagates to Next.js as a 500
**Fix:** Wrap `getUserMembership` call in a try/catch in `layout.tsx`; default to `{ tier: "free" }` on error

---

### BUG-3 (Critical): `/preferences` crashes with client-side TypeError

**URL:** `/en/preferences`
**Symptom:** `TypeError: o.patternPlaceholder.replace is not a function`
**Root Cause:**
- `app/[locale]/preferences/PreferencesClient.tsx:233`:
  ```tsx
  placeholder={mobileFilters.patternPlaceholder.replace("{type}", newType)}
  ```
- `useIntlayer()` in Next.js client components returns a React node/object, not a plain string
- Calling `.replace()` on a React node throws TypeError
**Fix:** Convert to string first: `String(mobileFilters.patternPlaceholder).replace("{type}", newType)`

### BUG-5 (Major): `/membership` fails to load membership data

**URL:** `/en/membership`
**Symptom:** "Failed to load membership information" after fetch to `/api/membership/status` returns `{"error":"User not found"}` with HTTP 404
**Root Cause:** Same as BUG-2 — `getUserMembership` queries `db.collection("user").findOne({ id: userId })` but the user document is not found. The `better-auth` MongoDB adapter may store users with `_id` (ObjectId) rather than an `id` string field.
**Fix:** Either align the query with how `better-auth` stores user documents, or auto-create the membership record on first access.

### BUG-6 (Minor): Sign-in email placeholder shows `[object Object]`

**URL:** `/en/sign-in`
**Symptom:** Both email input fields on the sign-in page show `[object Object]` as the placeholder text
**Root Cause:**
- `app/[locale]/sign-in/SignInClient.tsx:158,199`:
  ```tsx
  placeholder={emailPlaceholder}
  ```
- `emailPlaceholder` from `useIntlayer()` returns a React node/object, not a plain string
- When used as an HTML `placeholder` attribute, it serializes to `[object Object]`
**Fix:** Use `String(emailPlaceholder)` or extract the string value

### BUG-7 (Minor): Userscript `ReferenceError: Y0 is not defined`

**Symptom:** Console error on every page: `ReferenceError: Y0 is not defined at Window.<anonymous> (Lianki.user.js:1063:23)`
**Root Cause:** The deployed userscript (in browser extension context) has a scoping issue where `Y0` is referenced at line 1063 but is out of scope at that point. Local file defines `Y0` at line 50 and references it at line 1059 as `window.LiankiDeps = Y0` — the deployed version has a different line count suggesting minification changed variable scoping.
**Fix:** Review userscript bundling — ensure `Y0` / `LiankiDeps` export is in the same scope as its definition.
